## Теория

### Что это такое

До сих пор твой код, скорее всего, выполнялся **сверху вниз, строка за строкой**. Это называется **линейное выполнение**. Но реальные программы редко такие простые: им нужно **принимать решения** и **повторять действия**.

Для этого в Python есть два фундаментальных механизма:

1. **Условные конструкции** — «если… то… иначе…»
2. **Циклы** — «повтори это N раз, пока условие истинно»

---

#### Условия: `if`, `elif`, `else`

**Условие** — это способ сказать программе: «проверь что-то, и в зависимости от результата выбери, какой код выполнить».

Базовая форма:

```python
if условие:
    # код, если условие True
```

- После `if` идёт **выражение**, которое Python оценивает как `True` или `False`.
- Двоеточие `:` в конце — **обязательно**.
- Блок кода под условием **сдвигается вправо (отступ)** — обычно 4 пробела. В Python отступы — это не «красота», а **синтаксис**.

Полная форма с ветвлением:

```python
if условие_1:
    # выполнится, если условие_1 — True
elif условие_2:
    # выполнится, если условие_1 — False, а условие_2 — True
else:
    # выполнится, если все условия выше — False
```

- `elif` = **else if** («иначе, если»). Может быть сколько угодно.
- `else` — необязательный «запасной» вариант, когда ничего не подошло.

**Вложенные условия** — `if` внутри `if`:

```python
if user_is_logged_in:
    if user_is_admin:
        print("Панель администратора")
    else:
        print("Личный кабинет")
else:
    print("Войдите в систему")
```

#### Циклы: `for` и `while`

**Цикл** — повторение блока кода.

##### Цикл `for`

Используется, когда ты **знаешь заранее**, по чему итерироваться: по списку, строке, диапазону чисел.

```python
for элемент in последовательность:
    # тело цикла — выполняется для каждого элемента
```

Самый частый паттерн с числами — функция **`range()`**:

```python
range(5)        # 0, 1, 2, 3, 4
range(2, 7)     # 2, 3, 4, 5, 6  (от 2 до 6 включительно)
range(0, 10, 2) # 0, 2, 4, 6, 8  (шаг 2)
```

`range()` **не создаёт список в памяти** (в Python 3) — это «ленивый» генератор чисел. Для Junior достаточно понимать: `for i in range(5)` даст `i` от 0 до 4.

##### Цикл `while`

Используется, когда нужно повторять **пока условие истинно**, и заранее неизвестно, сколько итераций будет.

```python
while условие:
    # тело цикла
```

Классика — «угадай число», чтение ввода до корректного значения, ожидание события.

**Опасность:** если условие **никогда** не станет `False`, получится **бесконечный цикл** — программа зависнет.

---

#### `break`, `continue`, `pass`

| Ключевое слово | Что делает |
|----------------|------------|
| `break` | Немедленно **выходит** из цикла |
| `continue` | **Пропускает** оставшуюся часть текущей итерации и переходит к следующей |
| `pass` | **Ничего не делает** — заглушка, когда синтаксис требует блок кода, а логики пока нет |

```python
for i in range(10):
    if i == 5:
        break       # цикл остановится на i=5
    print(i)        # напечатает 0,1,2,3,4

for i in range(5):
    if i == 2:
        continue    # пропустит print при i=2
    print(i)        # 0, 1, 3, 4
```

---

#### `else` у циклов (редко, но знай)

У `for` и `while` может быть блок `else`, который выполнится, если цикл **завершился нормально** (без `break`):

```python
for n in range(3):
    print(n)
else:
    print("Цикл закончился без break")
```

На уровне Junior это не обязательно использовать каждый день, но при чтении чужого кода встретишь.

### Как это работает «под капотом»

#### Как Python оценивает условие

Любое выражение в `if` проходит через **`bool()`**:

```python
if 42:          # bool(42) → True
if 0:           # bool(0) → False
if "":          # bool("") → False
if "hello":     # bool("hello") → True
if None:        # bool(None) → False
if [1, 2]:      # bool([1,2]) → True (непустой список)
if []:          # bool([]) → False (пустой список)
```

Это называется **truthy** (считается истинным) и **falsy** (считается ложным).

**Falsy-значения в Python** (запомни этот список):

| Значение | Почему falsy |
|----------|--------------|
| `False` | Логическая ложь |
| `None` | Отсутствие значения |
| `0`, `0.0` | Нулевые числа |
| `""` | Пустая строка |
| `[]`, `()`, `{}` | Пустые коллекции |

Всё остальное — **truthy**.

#### Как выполняется `if` на уровне интерпретора

1. Python вычисляет выражение после `if`.
2. Вызывает `bool(результат)`.
3. Если `True` — выполняет блок с отступом.
4. Если `False` — **пропускает** блок и идёт к `elif` / `else` / следующей строке на том же уровне отступа.

**Короткое замыкание (short-circuit)** у `and` / `or`:

```python
# and — если первое False, второе даже не вычисляется
False and print("не напечатается")

# or — если первое True, второе не вычисляется
True or print("не напечатается")
```

Это важно, когда второе условие может вызвать ошибку:

```python
items = []
if len(items) > 0 and items[0] == "admin":  # безопасно
    pass

# Плохо (упадёт с IndexError):
# if items[0] == "admin" and len(items) > 0
```

#### Как работает `for`

1. Python берёт **итератор** из объекта (`iter(последовательность)`).
2. На каждой итерации вызывает `next(итератор)` — получает следующий элемент.
3. Когда элементы кончились — выбрасывается исключение `StopIteration`, цикл завершается.

`range(5)` внутри тоже итерируемый объект — поэтому `for i in range(5)` работает без создания списка `[0,1,2,3,4]`.

#### Как работает `while`

1. Проверяется условие → `bool(условие)`.
2. Если `True` — выполняется тело, возврат к шагу 1.
3. Если `False` — выход из цикла.

**Критично:** в теле `while` должно быть что-то, что **рано или поздно** сделает условие `False`. Иначе — бесконечный цикл.

### Где это полезно и применяется в реальной разработке

#### Условия — везде, где есть логика решений

| Сценарий | Пример |
|----------|--------|
| Авторизация | `if user.role == "admin":` |
| Валидация формы | `if len(password) < 8:` |
| Ценообразование | `if total > 1000: apply_discount()` |
| Обработка HTTP | `if response.status_code == 200:` |
| Парсинг данных | `if line.startswith("#"): continue` |

Без `if` не существует ни одного полезного приложения.

#### Цикл `for` — обработка коллекций

- Пройти по списку заказов и посчитать сумму.
- Отправить email каждому пользователю из списка.
- Прочитать файл построчно: `for line in file:`.
- Отрисовать 10 карточек товара на странице.

**Правило большого пальца:** есть список/диапазон/строка — бери `for`.

#### Цикл `while` — ожидание и интерактив

- Меню CLI: «пока пользователь не выбрал "выход"».
- Игра: «пока здоровье > 0».
- Retry-логика: «пока запрос к API неуспешен и попыток < 3».
- Чтение ввода: «пока не введут число».

**Правило:** неизвестно количество итераций заранее — смотри на `while`.

### Микро-моменты, нюансы, частые ошибки новичков

#### 1. Забытый отступ — `IndentationError`

```python
if x > 0:
print("положительное")   # ОШИБКА! нет отступа
```

В Python **нет** фигурных скобок `{}` для блоков — только отступы.

#### 2. Двоеточие забыли — `SyntaxError`

```python
if x > 0    # ОШИБКА — нужно if x > 0:
    print("ok")
```

#### 3. Сравнение вместо присваивания

```python
if x = 5:   # SyntaxError — нужно ==
if x == 5:  # правильно
```

#### 4. Сравнение цепочкой (фишка Python!)

В Python можно писать красиво:

```python
age = 25
if 18 <= age < 65:      # как в математике!
    print("Трудоспособный возраст")
```

Это эквивалентно `age >= 18 and age < 65`, но читается лучше.

#### 5. `if` без `else` — это нормально

Не каждое условие требует `else`. Если «иначе ничего не делать» — просто не пиши `else`.

#### 6. Вложенность — не увлекайся

Три уровня `if` внутри `if` — уже трудно читать. Лучше:

- Ранний выход (`return`, `continue`)
- Объединение условий через `and`/`or`
- Вынесение в отдельную функцию (позже)

#### 7. Off-by-one (ошибка «на единицу») с `range`

```python
for i in range(1, 11):   # 1..10 — 10 итераций
    print(i)

for i in range(10):      # 0..9 — тоже 10 итераций, но с нуля!
    print(i)
```

Спроси себя: «мне нужно 10 раз или числа от 1 до 10?»

#### 8. Изменение списка во время `for` по нему — ловушка

```python
nums = [1, 2, 3, 4, 5]
for n in nums:
    if n % 2 == 0:
        nums.remove(n)   # опасно! пропустит элементы
```

На Junior пока **не удаляй** из списка, по которому итерируешься. Создай новый список или итерируй копию. Подробнее — в главе про списки.

#### 9. `while True` + `break` — нормальный паттерн

```python
while True:
    command = input("> ")
    if command == "quit":
        break
    # обработка команды
```

Бесконечный цикл с явным выходом — стандарт в CLI и играх.

#### 10. Тернарный оператор (компактный `if`)

Когда нужно выбрать одно из двух значений в одну строку:

```python
status = "взрослый" if age >= 18 else "ребёнок"
```

Это **не** замена полноценному `if/elif/else` с блоками кода — только для простого выбора значения.

#### 11. `match` / `case` (Python 3.10+)

Аналог `switch` из других языков. На Junior полезно знать, что существует:

```python
match command:
    case "start":
        print("Старт")
    case "stop":
        print("Стоп")
    case _:
        print("Неизвестная команда")
```

В домашке пока используй классический `if/elif/else` — это база.

### Пример 1: Простой `if` / `elif` / `else`

```python
score = 73   # баллы за тест (от 0 до 100)

if score >= 90:
    grade = "A"           # отлично
elif score >= 75:
    grade = "B"           # хорошо
```

### Пример 5: `for` по списку — практическая задача

```python
prices = [199.99, 49.50, 1200.00, 15.00, 89.90]

total = 0
expensive_count = 0              # счётчик товаров дороже 100

for price in prices:
    total += price
    if price > 100:              # условие внутри цикла — нормально и часто

average = total / len(prices)    # средняя цена

print(f"Всего товаров: {len(prices)}")
print(f"Общая сумма: {total:.2f} руб.")
print(f"Средняя цена: {average:.2f} руб.")
print(f"Дорогих (>100 руб.): {expensive_count}")
```

**Вывод консоли:**

```
Всего товаров: 5
Общая сумма: 1554.39 руб.
Средняя цена: 310.88 руб.
Дорогих (>100 руб.): 2
```

### Пример 6: Цикл `while` — угадай число (упрощённо)

```python
secret = 7                       # загаданное число (в реале — random)
attempts = 0                     # счётчик попыток
max_attempts = 3                 # максимум попыток

# Имитируем ввод пользователя списком (чтобы пример был автономным)
guesses = [5, 9, 7]              # пользователь вводит 5, потом 9, потом 7
guess_index = 0

while attempts < max_attempts:
    guess = guesses[guess_index] # в реале: guess = int(input("Ваш вариант: "))
    guess_index += 1
    attempts += 1

    print(f"Попытка {attempts}: вы ввели {guess}")

    if guess == secret:
        print("🎉 Угадали!")
        break                    # выходим из цикла — дальше не играем
    elif guess < secret:
        print("Больше!")
    else:
        print("Меньше!")
else:
    # этот else сработает только если break НЕ вызывался
    print(f"Попытки кончились. Загаданное число было {secret}")
```

**Вывод консоли:**

```
Попытка 1: вы ввели 5
Больше!
Попытка 2: вы ввели 9
Меньше!
Попытка 3: вы ввели 7
🎉 Угадали!
```

### Пример 8: Меню с `while True` — реальный паттерн CLI

```python
balance = 1000.0                 # баланс счёта

while True:                      # бесконечный цикл
    print("\n--- Банкомат ---")
    print("1. Показать баланс")
    print("2. Снять деньги")
    print("3. Выход")

    choice = "2"                 # имитация ввода (в реале: input("Выбор: "))

    if choice == "1":
        print(f"Ваш баланс: {balance:.2f} руб.")

    elif choice == "2":
        amount = 300.0           # имитация ввода суммы
        if amount > balance:
            print("Недостаточно средств!")
        elif amount <= 0:
            print("Сумма должна быть положительной!")
        else:
            balance -= amount
            print(f"Снято {amount:.2f} руб. Остаток: {balance:.2f} руб.")

    elif choice == "3":
        print("До свидания!")
        break                    # единственный выход из while True

    else:
        print("Неверный пункт меню. Попробуйте снова.")

    break                        # убираем в реальном коде! здесь — чтобы пример не зациклился
```

**Вывод консоли:**

```

--- Банкомат ---
1. Показать баланс
2. Снять деньги
3. Выход
Снято 300.00 руб. Остаток: 700.00 руб.
```

*(В примере один проход — `break` в конце для демонстрации. В твоей домашке убери его, чтобы меню работало в цикле.)*

### Пример 9: Вложенные циклы — таблица умножения

### Пример 10: Комбо — FizzBuzz (легенда собеседований)

```python
# Для чисел 1–15:
# делится на 3 и 5 → "FizzBuzz"
# только на 3       → "Fizz"
# только на 5       → "Buzz"
# иначе             → само число

for n in range(1, 16):
    if n % 3 == 0 and n % 5 == 0:
        print("FizzBuzz")
```

### Таблицы истинности для `and`, `or`, `not`

Логика в коде — та же, что в жизни. Таблица показывает **все** комбинации:

#### `not` (отрицание)

| Исходное | `not` исходное |
|----------|----------------|
| `True`   | `False`        |
| `False`  | `True`         |

```python
print(not True)    # False
print(not False)   # True
print(not 0)       # True  (0 — falsy)
print(not 1)       # False (1 — truthy)
```

#### `and` (И — оба должны быть True)

| A | B | A `and` B |
|---|---|-----------|
| True  | True  | **True**  |
| True  | False | False     |
| False | True  | False     |
| False | False | False     |

```python
# Реальный пример: доступ к админке
is_logged_in = True
is_admin = False

if is_logged_in and is_admin:
    print("Админ-панель")
else:
    print("Доступ запрещён")   # сюда попадём
```

#### `or` (ИЛИ — достаточно одного True)

| A | B | A `or` B |
|---|---|----------|
| True  | True  | **True** |
| True  | False | **True** |
| False | True  | **True** |
| False | False | False    |

```python
# Выходной, если суббота ИЛИ воскресенье ИЛИ праздник
is_saturday = False
is_sunday = True
is_holiday = False

if is_saturday or is_sunday or is_holiday:
    print("Сегодня выходной!")
```

#### Комбинации — скобки задают приоритет

```python
# Без скобок: and «сильнее» or (как × сильнее + в математике)
result = True or False and False
# Сначала: False and False → False
# Потом:  True or False    → True
print(result)   # True

# Со скобками — явно говорим, что хотим
result = (True or False) and False
print(result)   # False
```

**Правило для Junior:** если в одном `if` больше двух `and`/`or` — **ставь скобки**, даже если Python и без них понял бы. Так читается легче.

### `for` или `while`? — дерево решений

```
Нужно повторить код?
│
├─ ДА → Есть готовая последовательность (список, строка, range)?
│        │
│        ├─ ДА → for
│        │
│        └─ НЕТ → Знаю точное число повторений?
│                 │
│                 ├─ ДА → for + range(n)
│                 │
│                 └─ НЕТ → while (пока условие)
│
└─ НЕТ → if (одноразовое решение)
```

| Ситуация | Что брать | Почему |
|----------|-----------|--------|
| Сумма списка цен | `for price in prices` | Есть коллекция |
| 10 раз спросить пароль | `for i in range(10)` | Фиксированное число попыток |
| Пока ввод невалидный | `while` | Не знаем, сколько раз введут плохо |
| Меню до «выход» | `while True` + `break` | Неизвестно, сколько команд |
| Обработка каждой строки файла | `for line in file` | (позже, в главе про файлы) |

**Типичная ошибка:** писать `while` с счётчиком, когда хватило бы `for`:

```python
# Работает, но не по-питоновски
i = 0
while i < 10:
    print(i)
    i += 1

# Лучше
for i in range(10):
    print(i)
```

### Пошаговый разбор: как Python выполняет код

Представь, что ты — интерпретатор. Идёшь строка за строкой.

#### Разбор 1: простой `if`

```python
x = 10
if x > 5:
    print("большое")
else:
    print("маленькое")
print("конец")
```

| Шаг | Строка | Что происходит | Результат |
|-----|--------|----------------|-----------|
| 1 | `x = 10` | В памяти: x → 10 | — |
| 2 | `if x > 5` | 10 > 5 → True | входим в блок if |
| 3 | `print("большое")` | Печать | `большое` |
| 4 | `else` | Пропускается (if уже сработал) | — |
| 5 | `print("конец")` | Печать | `конец` |

#### Разбор 2: цепочка `elif`

```python
score = 73
if score >= 90:
    grade = "A"
elif score >= 75:
    grade = "B"
elif score >= 60:
    grade = "C"
else:
    grade = "F"
```

| Шаг | Проверка | Результат | Действие |
|-----|----------|-----------|----------|
| 1 | `73 >= 90` | False | идём дальше |
| 2 | `73 >= 75` | False | идём дальше |
| 3 | `73 >= 60` | **True** | `grade = "B"`, **остальные elif не проверяются** |

#### Разбор 3: цикл `for` с `break`

```python
for i in range(1, 5):
    if i == 3:
        break
    print(i)
print("готово")
```

| Итерация | i | `i == 3`? | Действие |
|----------|---|-----------|----------|
| 1 | 1 | False | print(1) |
| 2 | 2 | False | print(2) |
| 3 | 3 | **True** | break — выход из цикла |
| — | — | — | print("готово") |

**Вывод:** `1`, `2`, `готово`. Числа 3 и 4 **не** печатаются.

#### Разбор 4: `while` — следи за изменением условия

```python
count = 0
while count < 3:
    print(count)
    count += 1
print("цикл завершён")
```

| Итерация | count до | `count < 3`? | print | count после |
|----------|----------|--------------|-------|-------------|
| 1 | 0 | True | 0 | 1 |
| 2 | 1 | True | 1 | 2 |
| 3 | 2 | True | 2 | 3 |
| 4 | 3 | **False** | — | цикл стоп |

**Вывод:** `0`, `1`, `2`, `цикл завершён`.

Если забыть `count += 1` — `count` всегда 0, условие всегда True → **бесконечный цикл**.

### FAQ начинающего (частые вопросы)

**В: Можно ли писать `if` в одну строку?**

Да, для простых случаев:

```python
if age >= 18: print("ok")   # работает, но для Junior — лучше многострочно
```

**В: Сколько `elif` можно писать?**

Сколько угодно. Но если веток больше 5–7 — подумай о `match/case` или словаре (позже).

**В: `while` быстрее `for`?**

На уровне Junior — **не думай о скорости**. Пиши читаемо. `for` по коллекции почти всегда понятнее.

**В: Что если мне нужен цикл, но я не знаю, сколько итераций, и нет while?**

Всегда есть `while`. Или `for` с `break` внутри — но `while` честнее.

**В: `range(10)` включает 10?**

**Нет.** `range(10)` → 0,1,2,...,9. Верхняя граница **не включается**. Это самый частый вопрос новичков.

**В: Почему мой `if` всегда срабатывает?**

Скорее всего, в условии **непустая строка** или **ненулевое число** — они truthy:

```python
x = 5
if x:          # всегда True, пока x не 0
    print("да")
```

**В: Можно ли использовать `and` вместо вложенного `if`?**

Часто да:

```python
# Вместо:
if a:
    if b:
        do()

# Можно:
if a and b:
    do()
```

Но если в каждой ветке **разный код** — вложенность или `elif` уместнее.

### Пример 18: Полная мини-программа — всё вместе

```python
"""
Мини-программа: регистрация на мероприятие.
Демонстрирует: if/elif, while, for, break, continue, флаги, валидацию.
"""

MIN_AGE = 18
MAX_PARTICIPANTS = 5

registered = []                        # список имён (пока просто список строк)
running = True

while running:
    print("\n=== Регистрация ===")
    print("1. Зарегистрировать участника")
    print("2. Показать список")
    print("3. Статистика")
    print("4. Выход")

    choice = "1"                       # имитация; в реале: input("Выбор: ")

    if choice == "1":
        if len(registered) >= MAX_PARTICIPANTS:
            print(f"Мест нет! Максимум {MAX_PARTICIPANTS} человек.")
            continue                   # возврат в начало while, не выходим

        name = "Иван"                  # имитация input("Имя: ")
        age_str = "20"                 # имитация input("Возраст: ")

        if not age_str.isdigit():
            print("Возраст должен быть числом!")
            continue

        age = int(age_str)
        if age < MIN_AGE:
            print(f"Минимальный возраст: {MIN_AGE} лет")
            continue

        registered.append(name)
        print(f"{name} ({age} лет) зарегистрирован!")

    elif choice == "2":
        if not registered:             # пустой список — falsy
            print("Список пуст")
        else:
            print("Участники:")
            for i, name in enumerate(registered, start=1):
                print(f"  {i}. {name}")

    elif choice == "3":
        adults = 0
        for _ in registered:           # _ — «переменная не нужна»
            adults += 1                # упрощённо: все взрослые
        print(f"Зарегистрировано: {len(registered)}/{MAX_PARTICIPANTS}")

    elif choice == "4":
        print("До встречи на мероприятии!")
        running = False                # выход из while

    else:
        print("Неверный пункт меню")

    break                              # УБЕРИ в своей версии! Для демо — один проход
```

**Что здесь важно:** один `while` управляет программой, внутри — `if/elif` для меню, `continue` для «мягких» ошибок, `for` для вывода списка, truthy/falsy для пустого списка.

### Задача 3 — Средняя
Дан список температур за неделю:

```python
temperatures = [22, 25, 19, 30, 28, 17, 24]
```

Напиши программу, которая:
1. Находит максимальную и минимальную температуру **без** `max()` и `min()`.
2. Считает среднюю температуру.
3. Считает, сколько дней было «жарко» (температура > 25).

**Подсказки:**
<details>
<summary>Намек 1</summary>

Паттерн «накопитель»: `max_temp = temperatures[0]`, потом `for t in temperatures:` и `if t > max_temp: max_temp = t`.

</details>
<details>
<summary>Намек 2</summary>

Средняя = сумма / количество. Сумму набирай в цикле. `len(temperatures)` — количество.

</details>
<details>
<summary>Намек 3</summary>

«Жарко» — паттерн «счётчик с условием»: `hot_days = 0`, в цикле `if t > 25: hot_days += 1`.

</details>
