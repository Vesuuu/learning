## Теория

### Что это такое

**Переменная** — это имя (метка), которое ты даёшь значению в памяти компьютера. Представь коробку с наклейкой: на наклейке написано `age`, а внутри лежит число `25`. В Python переменная создаётся простым присваиванием:

```python
age = 25
```

Никаких ключевых слов вроде `int age = 25` (как в C/Java) не нужно — Python сам понимает тип по значению.

**Тип данных** — это «вид» информации. Число, текст, логическое да/нет — у каждого вида свои правила: что можно делать, как хранится, что произойдёт при операциях.

Основные встроенные типы Python, с которыми ты будешь работать каждый день:

| Тип | Пример | Описание |
|-----|--------|----------|
| `int` | `42`, `-7`, `0` | Целые числа |
| `float` | `3.14`, `-0.5`, `2.0` | Дробные числа |
| `str` | `"Привет"`, `'мир'` | Строки (текст) |
| `bool` | `True`, `False` | Логические значения |
| `NoneType` | `None` | «Ничего», отсутствие значения |

**Операторы** — символы и команды, которые выполняют действия над данными: сложение, сравнение, проверка условий.

---

### Как это работает «под капотом»

Когда ты пишешь `x = 10`, Python делает примерно следующее:

1. Создаёт в памяти объект типа `int` со значением `10`.
2. Создаёт имя `x` в текущем пространстве имён (namespace) — словаре «имя → объект».
3. Связывает имя `x` с этим объектом.

Важный момент: **имя — это ссылка на объект**, а не «ячейка с данными» как в некоторых других языках.

```python
a = 10
b = a      # b теперь ссылается на тот же объект 10
a = 20     # a теперь ссылается на новый объект 20, b по-прежнему 10
```

Для **неизменяемых (immutable)** типов (`int`, `float`, `str`, `bool`) при «изменении» на самом деле создаётся **новый** объект, а имя переназначается на него. Старый объект остаётся, пока на него есть другие ссылки.

**Проверка типа** — функция `type()`:

```python
type(42)        # <class 'int'>
type(3.14)      # <class 'float'>
type("hello")   # <class 'str'>
```

**Преобразование типов (приведение)**:

```python
int("42")       # строка "42" → число 42
str(100)        # число 100 → строка "100"
float("3.14")   # строка → дробное число
bool(0)         # False (ноль — «ложь»)
bool(1)         # True
```

#### Арифметические операторы

| Оператор | Название | Пример | Результат |
|----------|----------|--------|-----------|
| `+` | Сложение | `5 + 3` | `8` |
| `-` | Вычитание | `5 - 3` | `2` |
| `*` | Умножение | `5 * 3` | `15` |
| `/` | Деление | `7 / 2` | `3.5` (всегда float!) |
| `//` | Целочисленное деление | `7 // 2` | `3` |
| `%` | Остаток от деления | `7 % 2` | `1` |
| `**` | Возведение в степень | `2 ** 3` | `8` |

Обрати внимание: оператор `/` **всегда** возвращает `float`, даже если результат целый: `10 / 2` → `5.0`, а не `5`.

#### Операторы сравнения

Возвращают `True` или `False`:

| Оператор | Значение | Пример |
|----------|----------|--------|
| `==` | Равно | `5 == 5` → `True` |
| `!=` | Не равно | `5 != 3` → `True` |
| `>` | Больше | `5 > 3` → `True` |
| `<` | Меньше | `5 < 3` → `False` |
| `>=` | Больше или равно | `5 >= 5` → `True` |
| `<=` | Меньше или равно | `3 <= 5` → `True` |

#### Логические операторы

Работают с `bool`:

| Оператор | Значение | Пример |
|----------|----------|--------|
| `and` | И (оба True) | `True and False` → `False` |
| `or` | ИЛИ (хотя бы один True) | `True or False` → `True` |
| `not` | НЕ (инверсия) | `not True` → `False` |

#### Операторы присваивания

| Оператор | Эквивалент | Пример |
|----------|------------|--------|
| `=` | простое присваивание | `x = 5` |
| `+=` | `x = x + 5` | `x += 5` |
| `-=` | `x = x - 5` | `x -= 5` |
| `*=` | `x = x * 5` | `x *= 5` |
| `/=` | `x = x / 5` | `x /= 5` |

---

### Где это полезно и применяется в реальной разработке

- **Переменные** — основа любого кода: имена пользователей, цены, статусы заказов, счётчики.
- **`int` и `float`** — расчёты: скидки, статистика, координаты, таймеры.
- **`str`** — всё текстовое: логины, сообщения, пути к файлам, JSON-поля.
- **`bool`** — флаги и условия: «авторизован ли пользователь», «оплачен ли заказ», «файл существует».
- **`None`** — «значение ещё не задано»: поле в БД пустое, функция ничего не вернула осознанно.
- **Операторы сравнения и логики** — основа `if/else`, фильтров, валидации форм, прав доступа.

Пример из реального API:

```python
user_age = 17
is_premium = False

if user_age >= 18 and is_premium:
    print("Доступ к контенту 18+")
else:
    print("Доступ запрещён")
```

### Микро-моменты, нюансы, частые ошибки новичков

**1. Путаница `=` и `==`**

- `=` — присваивание (положить значение в переменную).
- `==` — сравнение (равны ли два значения).

```python
x = 5       # правильно: присваиваем
x == 5      # это сравнение, вернёт True, но никуда не сохранится
if x = 5:   # ОШИБКА SyntaxError!
```

**2. Деление `/` vs `//`**

```python
7 / 2    # 3.5  — обычное деление
7 // 2   # 3    — целая часть (отбрасывает дробь)
7 % 2    # 1    — остаток
```

**3. Сложение числа и строки — TypeError**

```python
age = 25
print("Мне " + age + " лет")   # ОШИБКА!
print("Мне " + str(age) + " лет")  # правильно
print(f"Мне {age} лет")            # ещё лучше (f-string)
```

**4. Имена переменных**

Правила:
- Только буквы, цифры, `_`; не может начинаться с цифры.
- Регистр важен: `age` и `Age` — разные переменные.
- Нельзя использовать зарезервированные слова: `if`, `for`, `class`, `True`, `False`...

Стиль (PEP 8): `snake_case` — `user_name`, `total_price`, `is_active`.

**5. Float и точность (ты уже попался на диагностике!)**

Компьютер хранит дробные числа приблизительно:

```python
0.1 + 0.2 == 0.3   # False! (не True)
0.1 + 0.2          # 0.30000000000000004
```

Для денег и точных расчётов в продакшене используют `decimal.Decimal`, но на уровне Junior достаточно **знать**, что float — приблизительный.

**6. `input()` всегда возвращает строку**

```python
age = input("Введите возраст: ")  # это str, даже если ввели 25!
age = int(input("Введите возраст: "))  # правильно, если нужно число
```

**7. Truthy и Falsy**

В условиях `if` Python считает «ложными»:

```python
# Falsy (ложные):
bool(0), bool(0.0), bool(""), bool(None), bool([]), bool({})
# [] и {} — пустые списки/словари (подробно в гл. 4). Пока: «пустое = False».

# Truthy (истинные) — всё остальное:
bool(1), bool(-5), bool("hello"), bool([1, 2])
```

# Создаём переменные разных типов
name = "Алексей"          # str — строка с именем

### Пример 2: Арифметические операторы

```python
a = 17                    # первое число
b = 5                     # второе число

print("Сложение:", a + b)           # 17 + 5 = 22
print("Вычитание:", a - b)          # 17 - 5 = 12
print("Умножение:", a * b)          # 17 * 5 = 85
print("Деление:", a / b)            # 17 / 5 = 3.4 (float!)
print("Целочисленное деление:", a // b)  # 17 // 5 = 3
print("Остаток:", a % b)            # 17 % 5 = 2 (17 = 3*5 + 2)
print("Степень:", a ** b)           # 17^5 = 1419857
```

**Вывод консоли:**

```
Сложение: 22
Вычитание: 12
Умножение: 85
Деление: 3.4
Целочисленное деление: 3
Остаток: 2
Степень: 1419857
```

**Разбор:** `/` всегда даёт `float`, даже когда результат «целый». `//` отбрасывает дробную часть. `%` — остаток от деления: `17 = 3×5 + 2`. `**` — степень, растёт очень быстро.

### Пример 3: Операторы сравнения и логики

```python
x = 10
y = 20

# Сравнения — каждое выражение возвращает True или False
print(x == y)    # False — 10 не равно 20
print(x != y)    # True  — 10 не равно 20 (утверждение верно)
print(x < y)     # True  — 10 меньше 20
print(x >= 10)   # True  — 10 больше или равно 10

# Логические операторы — комбинируют условия
has_ticket = True
has_passport = False

# and — оба условия должны быть True
can_fly = has_ticket and has_passport
print("Может лететь:", can_fly)   # False — нет паспорта

# or — достаточно одного True
is_weekend = True
is_holiday = False
day_off = is_weekend or is_holiday
print("Выходной:", day_off)       # True — суббота/воскресенье

# not — инвертирует значение
print(not True)   # False
print(not False)  # True
```

**Вывод консоли:**

```
False
True
True
True
Может лететь: False
Выходной: True
False
True
```

**Разбор:** сравнения возвращают `bool`. `and` требует оба `True`; `or` — хотя бы один. `not` переворачивает значение. Это основа условий `if` в следующей главе.

### Пример 4: Приведение типов и input()

```python
# input() всегда возвращает строку
user_input = "1998"           # имитируем ввод пользователя
year = int(user_input)        # преобразуем str → int
current_year = 2026
age = current_year - year     # теперь можно вычитать числа

print(f"Вам {age} лет")       # f-string подставляет переменные в строку

# Другие приведения типов
price = 19.99
price_str = str(price)        # float → str
print("Цена: " + price_str + " руб.")

quantity = 3
total = price * quantity      # float * int → float
print(f"Итого: {total} руб.")
```

**Вывод консоли:**

```
Вам 28 лет
Цена: 19.99 руб.
Итого: 59.97 руб.
```

**Разбор:** без `int()` нельзя вычитать год из строки. `str()` нужен для конкатенации `+` со строкой. `float * int` даёт `float` — Python «поднимает» тип.

### Пример 5: Составные операторы присваивания и None

```python
score = 100               # начальный счёт

score += 25               # score = score + 25 → 125
print("После бонуса:", score)

score -= 10               # score = score - 10 → 115
print("После штрафа:", score)

score *= 2                # score = score * 2 → 230
print("Удвоенный:", score)

# None — «пустое» значение
result = None             # функция ещё ничего не вернула
print(result)             # None
print(type(result))       # <class 'NoneType'>
print(result is None)     # True — правильная проверка на None
```

**Вывод консоли:**

```
После бонуса: 125
После штрафа: 115
Удвоенный: 230
None
<class 'NoneType'>
True
```

**Разбор:** `+=`, `-=`, `*=` — сокращённая запись. `None` — отдельный тип «значения нет». Проверять через `is None`, а не `== None` (хотя для новичка оба работают, `is` — идиоматично).

### Пример 6: Ловушка с float (запомни!)

```python
# Новички ожидают True — но получат False
print(0.1 + 0.2 == 0.3)       # False

# Посмотрим реальное значение
print(0.1 + 0.2)              # 0.30000000000000004

# Правильный способ сравнить float — с допуском (погрешностью)
result = 0.1 + 0.2
print(abs(result - 0.3) < 0.0001)   # True — «достаточно близко»
```

**Вывод консоли:**

```
False
0.30000000000000004
True
```

**Разбор:** float — двоичное приближение. Для точных денег используют `Decimal`, для Junior достаточно знать ловушку и сравнивать с допуском `abs(a - b) < epsilon`.

### Пример 7: f-strings — современный способ форматирования

```python
product = "Ноутбук"
price = 89990.5
quantity = 2

# f-string: буква f перед кавычками, переменные в {фигурных скобках}
line = f"Товар: {product}, цена: {price} руб., кол-во: {quantity}"
print(line)

# Можно выражения прямо внутри скобок
total = price * quantity
print(f"Итого: {total:.2f} руб.")   # :.2f — два знака после запятой

# Выравнивание и заполнение
code = 7
print(f"Заказ №{code:05d}")         # 00007 — пять цифр с нулями слева
```

**Вывод консоли:**

```
Товар: Ноутбук, цена: 89990.5 руб., кол-во: 2
Итого: 179981.00 руб.
Заказ №00007
```

**Разбор:** f-strings (Python 3.6+) — самый читаемый способ вставить переменные в текст. `{total:.2f}` форматирует число. `{code:05d}` — целое с ведущими нулями. Лучше, чем `"Цена: " + str(price)`.

### Пример 8: Truthy и Falsy в условиях ⏳ (превью гл. 2)

> Здесь уже есть `if` — **формально это глава 2**. Пока запомни: пустое/ноль = False. Подробно — в следующей главе.

```python
# Python не требует явного bool — «истинность» значения
name = ""           # пустая строка — Falsy
score = 0           # ноль — Falsy
items = [1, 2, 3]   # непустой список — Truthy

if name:
    print("Имя задано")
else:
    print("Имя пустое")       # сработает это

if score:
    print("Есть очки")
else:
    print("Очков нет")        # сработает это

# Практика: проверка ввода
user_input = input("Введите что-нибудь: ") if False else "привет"
if user_input:
    print(f"Вы ввели: {user_input}")
```

**Вывод консоли:**

```
Имя пустое
Очков нет
Вы ввели: привет
```

**Разбор:** в `if` Python вызывает `bool(значение)`. Пустые строки, нули, `None`, пустые списки — «ложь». Непустая строка `"привет"` — «истина». Удобно для проверки «есть ли данные».

### Пример 9: Имена переменных и snake_case

```python
# Правильно (PEP 8)
user_name = "Мария"
total_price = 1500
is_active = True

# Плохой стиль (но Python не запретит)
userName = "Иван"      # camelCase — не принято в Python
TOTAL = 100            # КАПС — обычно для констант модуля

# Ошибки — так нельзя:
# 2name = "Ошибка"     # SyntaxError: имя не может начинаться с цифры
# class = 5            # SyntaxError: class — зарезервированное слово

print(user_name, total_price, is_active)
print(f"Пользователь {user_name}, сумма {total_price}, активен: {is_active}")
```

**Вывод консоли:**

```
Мария 1500 True
Пользователь Мария, сумма 1500, активен: True
```

**Разбор:** `snake_case` — стандарт Python: слова через `_`. Регистр важен: `age` ≠ `Age`. Константы иногда пишут `MAX_SIZE`, но для обычных переменных — только строчные буквы.

### Пример 10: Приоритет операторов

```python
# Без скобок результат может удивить
result1 = 2 + 3 * 4       # сначала *, потом + → 2 + 12 = 14
result2 = (2 + 3) * 4     # скобки меняют порядок → 20

print("2 + 3 * 4 =", result1)
print("(2 + 3) * 4 =", result2)

# Сравнение и логика
age = 20
has_ticket = True
# and сильнее or? — and выполняется раньше
can_enter = age >= 18 and has_ticket or False
print("Может войти:", can_enter)

# Явные скобки — читаемее
can_enter_clear = (age >= 18) and has_ticket
print("Может войти (ясно):", can_enter_clear)
```

**Вывод консоли:**

```
2 + 3 * 4 = 14
(2 + 3) * 4 = 20
Может войти: True
Может войти (ясно): True
```

**Разбор:** `*`, `/` раньше `+`, `-`. Скобки `()` — лучший способ показать намерение. Для сложных условий разбивай на отдельные переменные `can_enter` — код понятнее.

### Пример 11: Множественное присваивание и обмен значений

```python
# Несколько имён — одно значение (осторожно с изменяемыми объектами!)
a = b = c = 0
print(a, b, c)    # 0 0 0

# Распаковка — несколько переменных за раз
x, y, z = 10, 20, 30
print(x, y, z)    # 10 20 30

# Обмен без временной переменной — фишка Python
first = "яблоко"
second = "груша"
first, second = second, first    # кортеж справа, распаковка слева
print(first, second)             # груша яблоко

# Возврат нескольких значений из «функции» (пока без def — идея)
min_val, max_val = 3, 9
print(f"От {min_val} до {max_val}")
```

**Вывод консоли:**

```
0 0 0
10 20 30
груша яблоко
От 3 до 9
```

**Разбор:** `a, b = b, a` — элегантный swap через кортеж. `x, y, z = 10, 20, 30` — распаковка. Это частый паттерн в Python-коде.

### Пример 12: type() vs isinstance() и проверка None

```python
value = 42
print(type(value) == int)           # True — сравнение типа
print(isinstance(value, int))       # True — предпочтительнее

text = "hello"
print(isinstance(text, str))        # True
print(isinstance(text, int))        # False

# isinstance умеет несколько типов
x = 3.14
print(isinstance(x, (int, float)))  # True — число int или float

# None — отдельная проверка
data = None
print(data is None)                 # True — именно None
print(data is not None)             # False
```

**Вывод консоли:**

```
True
True
True
False
True
True
False
```

**Разбор:** `isinstance(obj, тип)` гибче `type() ==`: учитывает наследование (пригодится в ООП). Для `None` используй `is None` — быстрее и однозначнее.

# FAQ

**Можно ли менять тип переменной?**  
Да. `x = 10` (int), потом `x = "текст"` (str) — имя просто ссылается на другой объект. Но в хорошем коде одна переменная обычно хранит один «смысл».

**Почему `10 / 2` даёт `5.0`, а не `5`?**  
В Python 3 оператор `/` **всегда** возвращает `float`. Нужно целое — используй `//`.

**Что лучше: `+` со `str()` или f-string?**  
f-string — короче и читаемее: `f"Мне {age} лет"` вместо `"Мне " + str(age) + " лет"`.

**Когда использовать `True`/`False`, а когда просто `if value`?**  
Если переменная уже `bool` — оба варианта OK. Для строк, чисел, списков удобнее `if name:` (проверка «не пусто»).

**Зачем `None`, если есть `0` или `""`?**  
`None` — «значение отсутствует намеренно». `0` — число ноль. `""` — пустая строка. Разные смыслы в API и БД.

**Как сравнивать float без сюрпризов?**  
`abs(a - b) < 0.0001` или модуль `math.isclose(a, b)`.

**Можно ли писать имена по-русски?**  
Технически да (`возраст = 25`), но в реальных проектах — только латиница и `snake_case`.
