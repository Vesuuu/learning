# ЧАСТЬ 1. Подсчёт частот

## 1.1. Зачем считать частоты

Представь список голосов на выборах: `["Anna", "Bob", "Anna", "Vika", "Anna", "Bob"]`.  
Вопрос: кто победил? Тебе нужно знать, **сколько раз** встретилось каждое имя.

Или строка `"aabbc"` — сколько раз каждая буква?  
`a → 2`, `b → 2`, `c → 1`.

**Частотный словарь** (frequency map) — это обычный `dict`, где:
- **ключ** — элемент (буква, число, слово),
- **значение** — сколько раз он встретился.

Такой словарь решает целый класс задач:
- анаграммы (одинаковые ли наборы букв),
- самый частый символ,
- дубликаты,
- «есть ли два числа с суммой X» (см. часть 3).

Сложность построения: **O(n)** — один проход по данным.

---

## 1.2. Аналогия

Ты пересчитываешь монеты в кошельке:
- 5 монет по 10 ₽,
- 3 монеты по 50 ₽,
- 1 монета 100 ₽.

Ты не запоминаешь каждую монету по отдельности — ты **группируешь по номиналу** и считаешь количество.  
Частотный словарь делает то же самое с буквами, словами или числами.

---

## 1.3. Ручной подсчёт через цикл

Самый понятный способ — без магии, только цикл и словарь:

```python
def count_letters(text):
    freq = {}
    for ch in text:
        if ch in freq:
            freq[ch] += 1
        else:
            freq[ch] = 1
    return freq
```

Короче через `.get()`:

```python
def count_letters(text):
    freq = {}
    for ch in text:
        freq[ch] = freq.get(ch, 0) + 1
    return freq
```

**`.get(ch, 0)`** — «дай значение по ключу `ch`, а если ключа нет — верни 0».  
Так не нужен `if ch in freq`.

---

## 1.4. `collections.Counter` — готовый счётчик

В стандартной библиотеке есть удобный класс:

```python
from collections import Counter

freq = Counter("aabbc")
print(freq)           # Counter({'a': 2, 'b': 2, 'c': 1})
print(freq["a"])      # 2
print(freq.most_common(1))  # [('a', 2)] — самый частый
```

`Counter` ведёт себя как словарь, но умеет:
- `most_common(n)` — n самых частых элементов,
- сложение счётчиков (для объединения),
- вычитание (для сравнения наборов).

На Junior-уровне: **умей посчитать руками** и **знай, что Counter есть** — в реальном коде часто используют его.

---

## 1.5. Типичные ошибки

| Ошибка | Что происходит | Как правильно |
|--------|----------------|---------------|
| Забыли `.get()` и пишут `freq[ch] += 1` для нового ключа | `KeyError` | `freq[ch] = freq.get(ch, 0) + 1` |
| Считают пробелы и регистр, когда не надо | `"A"` и `"a"` — разные ключи | `text.lower()` или фильтр `if ch.isalpha()` |
| Ищут максимум через сортировку всего словаря | Лишняя работа O(k log k) | `max(freq, key=freq.get)` или `most_common(1)` |

2. `freq.get("x", 0)` если `"x"` нет в словаре? → **0**.
3. `Counter("aaa")["a"]`? → **3**.

# ЧАСТЬ 2. Два указателя

## 2.1. Что это такое

**Два указателя** — два индекса (`left`, `right` или `i`, `j`), которые двигаются по строке или списку по **правилам задачи**.

Зачем:
- часто **один проход** O(n) вместо вложенных циклов O(n²),
- не нужна дополнительная память (или минимум памяти),
- код проще и быстрее.

Два главных варианта:

| Вариант | Как двигаются | Пример задачи |
|---------|---------------|---------------|
| **С разных концов** | `left` с начала, `right` с конца → к центру | палиндром, пара с суммой в отсортированном массиве |
| **В одну сторону** | оба слева направо, с разной скоростью | удаление дубликатов, фильтрация на месте |
| **Скользящее окно** | `left` и `right` задают отрезок фиксированной или переменной длины | максимум суммы k подряд идущих элементов |

---

## 2.2. Аналогия — с разных концов

Проверяешь, симметрично ли слово «топот»:
- смотришь на первую и последнюю букву — совпали?
- потом на вторую и предпоследнюю,
- и так до центра.

Это и есть два указателя: `left = 0`, `right = len(word) - 1`, сближаются.

---

## 2.3. Палиндром — классика двух указателей

Палиндром читается одинаково слева направо и справа налево: `"шалаш"`, `"level"`.

```python
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True
```

Сложность: **O(n)** — каждый символ смотрим максимум один раз.

**Важно:** в задачах часто просят игнорировать регистр и знаки препинания — тогда сначала нормализуешь строку (`lower()`, убрать не-буквы).

---

## 2.4. Пара с заданной суммой в отсортированном списке

Задача: в **отсортированном** списке есть ли два числа, дающие в сумме `target`?

Наивно — два вложенных цикла O(n²).  
С двумя указателями — O(n):

```python
def has_pair_sum_sorted(arr, target):
    left, right = 0, len(arr) - 1
    while left < right:
        current = arr[left] + arr[right]
        if current == target:
            return True
        if current < target:
            left += 1      # сумма мала — нужно большее число слева
        else:
            right -= 1     # сумма велика — уменьшаем правое
    return False
```

Почему работает: список отсортирован. Если сумма мала — увеличиваем левый элемент (следующий больше). Если велика — уменьшаем правый.

**Условие:** список должен быть отсортирован (как для бинарного поиска в главе 15).

---

## 2.5. Два указателя в одну сторону — убрать дубликаты

В **отсортированном** списке оставить только уникальные, сохранив порядок:

```python
def unique_sorted(arr):
    if not arr:
        return []
    result = [arr[0]]
    for i in range(1, len(arr)):
        if arr[i] != arr[i - 1]:
            result.append(arr[i])
    return result
```

Или «на месте» — два указателя `write` и `read` двигаются в одну сторону:

```python
def unique_inplace(arr):
    """arr должен быть отсортирован. Возвращает длину уникальной части."""
    if not arr:
        return 0
    write = 1
    for read in range(1, len(arr)):
        if arr[read] != arr[write - 1]:
            arr[write] = arr[read]
            write += 1
    return write

nums = [1, 1, 2, 2, 2, 3, 5, 5]
length = unique_inplace(nums)
print(nums[:length])
```

**Вывод:** `[1, 2, 3, 5]`

`read` перебирает все элементы, `write` — позиция, куда записываем следующий уникальный. Не создаём новый список — экономим память O(1) дополнительно.

---

## 2.6. Скользящее окно (кратко)

Окно — подряд идущий кусок массива длины `k`.  
`left` и `right` задают границы; при сдвиге `right` на 1 добавляем элемент, при сдвиге `left` — убираем.

Пример: максимальная сумма **k** подряд идущих чисел — за один проход O(n), без пересчёта всего окна с нуля каждый раз.

На Junior достаточно **понять идею**; в примерах ниже будет простой вариант.

---

### ✅ Проверь себя — часть 2

1. Палиндром `"abba"` — сколько сравнений пар символов? → **2** (`a/a`, `b/b`).
2. Два указателя с концов на отсортированном `[1,2,3,4,5]`, target=9 — найдут пару? → **Да, 4+5**.
3. `has_pair_sum_sorted` на неотсортированном списке надёжен? → **Нет**, сначала сортировка.

# ЧАСТЬ 3. Классические мини-задачи

## 3.1. Анаграмма

Два слова — анаграммы, если из одних и тех же букв можно составить оба: `"listen"` и `"silent"`.

**Способ 1:** отсортировать буквы и сравнить — O(n log n).

**Способ 2 (лучше):** сравнить частоты букв — O(n).

```python
from collections import Counter

def are_anagrams(a, b):
    return Counter(a.lower()) == Counter(b.lower())
```

Или без Counter — два словаря частот и сравнение.

---

## 3.2. Two Sum — два числа с суммой target (необязательно отсортированный список)

Задача: в списке найти **индексы** двух чисел, сумма которых равна `target`.

Два указателя с концов работают только на **отсортированном** массиве и не дают исходные индексы без доп. работы.

**Стандартный приём Junior:** словарь «число → индекс» за один проход O(n):

```python
def two_sum(nums, target):
    seen = {}
    for i, x in enumerate(nums):
        need = target - x
        if need in seen:
            return seen[need], i
        seen[x] = i
    return None
```

`need` — **дополнение**: чего не хватает до `target`.  
Если `need` уже встречался — пара найдена.

Память O(n) за словарь — нормальная плата за скорость.

---

## 3.3. Слияние двух отсортированных списков

Как в merge из сортировки слиянием: два указателя по началу каждого списка, берём меньший, двигаем его указатель.

```python
def merge_sorted(a, b):
    i, j = 0, 0
    result = []
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    result.extend(a[i:])
    result.extend(b[j:])
    return result
```

Сложность: **O(len(a) + len(b))** — каждый элемент обработан один раз.

---

## 3.4. Первый неповторяющийся символ

Строка `"aabbcdd"` → первый символ, который встречается **ровно один раз** — `'c'`.

1. Посчитать частоты — O(n).
2. Второй проход слева направо — найти первый с частотой 1 — O(n).

Итого O(n), два прохода (или один, если используешь OrderedDict — на Junior не обязательно).

---

## 3.5. Когда что выбирать

| Задача | Инструмент | Сложность |
|--------|------------|-----------|
| Сколько раз каждый элемент | `dict` / `Counter` | O(n) |
| Палиндром | два указателя с концов | O(n) |
| Пара с суммой, список отсортирован | два указателя с концов | O(n) |
| Пара с суммой, любой список, нужны индексы | словарь дополнений | O(n) |
| Анаграмма | `Counter` или частоты | O(n) |
| Два отсортированных списка в один | merge, два указателя | O(n) |
| Уникальные в отсортированном | один проход, сравнение соседей | O(n) |

# Практика — 12 примеров с разбором

## Пример 1. Частоты букв — ручной словарь

```python
def letter_freq(text):
    freq = {}
    for ch in text:
        freq[ch] = freq.get(ch, 0) + 1
    return freq

print(letter_freq("hello"))
```

**Вывод:** `{'h': 1, 'e': 1, 'l': 2, 'o': 1}`

**Разбор:** один проход по строке. Каждая буква — ключ, значение увеличивается на 1. `.get(ch, 0)` защищает от `KeyError` для новой буквы.

## Пример 2. `Counter` и `most_common`

```python
from collections import Counter

votes = ["Anna", "Bob", "Anna", "Anna", "Vika", "Bob"]
c = Counter(votes)
print(c)
print(c.most_common(1))
```

**Вывод:**
```
Counter({'Anna': 3, 'Bob': 2, 'Vika': 1})
[('Anna', 3)]
```

**Разбор:** `most_common(1)` возвращает список из одной пары `(элемент, количество)` — победитель голосования.

## Пример 3. Самая частая буква в слове

```python
from collections import Counter

def most_common_char(word):
    if not word:
        return None
    c = Counter(word.lower())
    ch, count = c.most_common(1)[0]
    return ch, count

print(most_common_char("mississippi"))
```

**Вывод:** `('i', 4)`

**Разбор:** в `"mississippi"` чаще всего встречается `i` (4 раза). `lower()` чтобы не путать регистры.

## Пример 4. Палиндром — два указателя

```python
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True

print(is_palindrome("level"))
print(is_palindrome("python"))
```

**Вывод:**
```
True
False
```

**Разбор:** `"level"`: сравниваем `l/l`, `e/e`, `v/v` — всё совпало. `"python"`: `p` ≠ `n` на первой же паре → `False`. Сложность O(n).

## Пример 5. Палиндром с игнорированием не-букв

```python
def clean(s):
    return "".join(ch.lower() for ch in s if ch.isalpha())

def is_palindrome_phrase(s):
    t = clean(s)
    return t == t[::-1]

print(is_palindrome_phrase("A man, a plan, a canal: Panama"))
```

**Вывод:** `True`

**Разбор:** сначала оставляем только буквы в нижнем регистре → `"amanaplanacanalpanama"`. Сравнение с перевёрнутой строкой — тоже валидный O(n) способ (альтернатива двум указателям).

## Пример 6. Есть ли пара с суммой в отсортированном списке

```python
def has_pair_sum(arr, target):
    left, right = 0, len(arr) - 1
    while left < right:
        s = arr[left] + arr[right]
        if s == target:
            return True, left, right
        if s < target:
            left += 1
        else:
            right -= 1
    return False, -1, -1

arr = [1, 2, 4, 8, 10]
print(has_pair_sum(arr, 14))
print(has_pair_sum(arr, 7))
```

**Вывод:**
```
(True, 2, 4)
(False, -1, -1)
```

**Разбор:** `14 = 4 + 10` → `arr[2]=4`, `arr[4]=10`, индексы 2 и 4. Для `7` пары нет (1+2=3, 2+4=6, 4+8=12 — ни одна пара не даёт 7). Список **отсортирован** — без этого алгоритм неверен.

## Пример 7. Уникальные элементы в отсортированном списке

```python
def unique_in_sorted(arr):
    if not arr:
        return []
    out = [arr[0]]
    for i in range(1, len(arr)):
        if arr[i] != arr[i - 1]:
            out.append(arr[i])
    return out

print(unique_in_sorted([1, 1, 2, 2, 2, 3, 5, 5]))
```

**Вывод:** `[1, 2, 3, 5]`

**Разбор:** дубликаты в отсортированном списке стоят **рядом**. Сравниваем с предыдущим — один проход O(n).

def are_anagrams(w1, w2):

**Вывод:**

True

## Пример 9. Two Sum — индексы двух чисел

```python
def two_sum(nums, target):
    seen = {}
    for i, x in enumerate(nums):
        need = target - x
        if need in seen:
            return seen[need], i
        seen[x] = i
    return None

print(two_sum([2, 7, 11, 15], 9))
print(two_sum([3, 2, 4], 6))
```

**Вывод:**
```
(0, 1)
(1, 2)
```

**Разбор:** для `9` на индексе 1 число `7`, нужно `2` — оно было на индексе 0. Словарь `seen` хранит «уже видели → индекс». Один проход O(n).

## Пример 10. Слияние двух отсортированных списков

```python
def merge(a, b):
    i = j = 0
    res = []
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            res.append(a[i])
            i += 1
        else:
            res.append(b[j])
            j += 1
    res.extend(a[i:])
    res.extend(b[j:])
    return res

print(merge([1, 3, 5], [2, 4, 6]))
```

**Вывод:** `[1, 2, 3, 4, 5, 6]`

**Разбор:** всегда берём меньший из «голов» списков — как при сортировке слиянием. Остаток дописываем в конец.

## Пример 11. Первый неповторяющийся символ

```python
def first_unique_char(s):
    freq = {}
    for ch in s:
        freq[ch] = freq.get(ch, 0) + 1
    for i, ch in enumerate(s):
        if freq[ch] == 1:
            return ch, i
    return None, -1

print(first_unique_char("aabbcdd"))
print(first_unique_char("aabb"))
```

**Вывод:**
```
('c', 4)
(None, -1)
```

**Разбор:** сначала частоты, потом первый символ с частотой 1. В `"aabbcdd"` это `'c'` на индексе 4. В `"aabb"` уникальных нет.

## Пример 12. Скользящее окно — максимальная сумма k элементов подряд

```python
def max_sum_k(arr, k):
    if len(arr) < k:
        return None
    window_sum = sum(arr[:k])
    best = window_sum
    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]
        if window_sum > best:
            best = window_sum
    return best

print(max_sum_k([2, 1, 5, 1, 3, 2], 3))
```

**Вывод:** `9`

**Разбор:** окно длины 3. Суммы окон: `[2,1,5]=8`, `[1,5,1]=7`, `[5,1,3]=9`, `[1,3,2]=6`. Максимум 9.  
При сдвиге окна не пересчитываем все k элементов — вычитаем ушедший слева, прибавляем новый справа. O(n).

# Шпаргалка

| Паттерн | Суть | Сложность |
|---------|------|-----------|
| Частотный словарь | `freq[x] = freq.get(x, 0) + 1` | O(n) |
| `Counter` | готовый счётчик + `most_common` | O(n) |
| Два указателя с концов | `left`, `right` сближаются | O(n) |
| Пара суммы (sorted) | увеличиваем `left` или уменьшаем `right` | O(n) |
| Two Sum (индексы) | словарь `need in seen` | O(n) |
| Merge sorted | два указателя по началам | O(n) |
| Скользящее окно | сумма окна обновляется за O(1) | O(n) |

# FAQ

**Counter или ручной dict?**  
На собеседовании покажи ручной цикл. В проекте — `Counter` короче и читаемее.

**Зачем два указателя, если есть `set`?**  
Для **палиндрома** и **пары в отсортированном** массиве указатели дают O(n) времени и O(1) памяти (без копии).

**Two Sum и два указателя — это одно и то же?**  
Название похожее, но **Two Sum** обычно решают **словарём**. Два указателя с концов — другая задача (пара в отсортированном массиве).

**Нужно ли учить скользящее окно глубоко?**  
На Junior — базовое окно фиксированной длины (как в примере 12). Переменное окно — позже.

**Палиндром: `s == s[::-1]` или указатели?**  
Оба O(n). Указатели — классика для собеседований; срез — быстрее написать в коде.

**Что если в Two Sum одно число используется дважды?**  
Условие обычно требует **два разных индекса**. Поэтому сначала проверяем `need in seen`, потом добавляем текущее число — иначе `[3], target=6` с одной тройкой не сработает.

**Можно ли two_sum без словаря?**  
Да, но вложенные циклы — O(n²). На собеседовании словарь — ожидаемый ответ.

**Зачем merge двух списков?**  
Это кирпичик сортировки слиянием и задача «объединить два отсортированных потока данных» (логи, очереди).
